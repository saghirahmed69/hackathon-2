## Prompt

/sp.specify
Build a Phase I Todo application as a Python console program.

This is an in-memory, single-user Todo application intended as Phase I of the
"Hackathon II ‚Äì Evolution of Todo" project.

The application must allow a user to manage a list of todo tasks using a
command-line interface.

Functional requirements:
- Users can add a task with a title and optional description
- Users can view all tasks with their ID, title, description, and completion status
- Users can update an existing task's title or description
- Users can delete a task by ID
- Users can mark a task as complete or incomplete

Behavior requirements:
- All tasks are stored in memory only (no database, no file persistence)
- Each task has a unique numeric ID generated by the system
- The application runs in a loop until the user chooses to exit
- The console interface presents a clear menu of options
- Invalid inputs (non-existent task IDs, invalid menu choices) are handled gracefully

Constraints:
- No authentication
- No external libraries
- No networking or UI frameworks
- Code must be generated exclusively by Claude Code
- All behavior must be traceable to this specification

Non-functional requirements:
- Clean, readable Python structure
- Functions should have single responsibility
- Output should be clear and user-friendly

This Phase I implementation serves as the foundation for later web, AI,
and cloud-native phases.
